| WN11-AU-000500   | The Application event log size must be configured to 32768 KB or greater | Inadequate log size will cause the log to fill up quickly. This may prevent audit events from being recorded properly and require frequent attention by administrative personnel | PowerShell | [View Remediation](https://github.com/stowett/DISA-STIGS-Scripts/blob/main/WN11-AU-000500.ps1)|
| WN11-CC-000110   | Printing over HTTP must be prevented  | This setting blocks Windows 11 from allowing print jobs over HTTP (port 80). In other words, it prevents users or systems from sending print jobs through an unencrypted web connection. Instead, secure methods like HTTPS or other protected protocols should be used |PowerShell | [View Remediation]                                         |
| WN11-CC-000285   |  The Remote Desktop Session Host must require secure RPC communications  |This setting forces Remote Desktop Services (RDS) to use secure RPC (Remote Procedure Call) communication. In simple terms, when someone connects via Remote Desktop, the system ensures the communication channel is encrypted and authenticated, not open or weak, thus preventing man-in-the-middle attacks | PowerShell |[View Remediation]  
| WN11-AU-000560   |  Windows 11 must be configured to audit other Logon/Logoff Events Successes  | This setting ensures that Windows logs successful “other” logon/logoff-related events. It tells Windows to record additional authentication-related activities that don’t fall into the basic logon/logoff categories such as: Special logon events (e.g., elevated privileges), Session reconnections/disconnections, and other authentication behaviors. Logon events are essential to understanding user activity and detecting potential attacks   | PowerShell |[View Remediation]  
| WN11-AU-000584   | Windows 11 must be configured to audit handle manipulation successes | This setting tracks when a process interacts, or rather, “handles” system objects like files, registry keys, or processes. A handle is basically a reference or pointer that a program uses to access a resource. Handle manipulation is often used in: Privilege escalation attacks, Process injection (malware hiding inside legit apps) and Unauthorized access to sensitive system objects | PowerShell |[View Remediation]  
| WN11-CC-000345   |  The Windows Remote Management (WinRM) service must not use Basic authentication  |This setting prevents WinRM from using weak log-in methods such as the use of plain text passwords that could be used to compromise a system. WinRM is used for: Remote PowerShell (administration) and Automation tools (like scripts, management systems)| PowerShell |[View Remediation]  
| WN11-UR-000070   |  The "Deny access to this computer from the network" user right on workstations must be configured to prevent access from highly privileged domain accounts and local accounts on domain systems and unauthenticated access on all systems  | This setting controls who is NOT allowed to access a computer over the network. It blocks certain high-risk accounts from connecting remotely to a workstation including: highly privileged domain accounts (like Domain Admins), local accounts on domain-joined systems and unauthenticated users (anonymous access)  | PowerShell |[View Remediation]  
| WN11-PK-000005   |  The DoD Root CA certificates must be installed in the Trusted Root Store | This requirement ensures that DoD (Department of Defense) Root Certificate Authorities (CAs) are installed in Windows 11’s. | WN11-CC-000204   |  Enhanced diagnostic data must be limited to the minimum required to support Windows Analytics  | This setting controls how much diagnostic and telemetry data Windows 11 sends to Microsoft (or enterprise analytics systems).Some features may communicate with the vendor, sending system information or downloading data or components for the feature. Limiting this capability will prevent potentially sensitive information from being sent outside the enterprise.  | PowerShell |[View Remediation]
